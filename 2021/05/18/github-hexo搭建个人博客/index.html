github+hexo搭建个人博客

Author: John Doe Date: May 18, 2021  9:53:05

背景

准研究生一枚，导师要求在暑假学习一些课程并且要在github上维护一个自己的博客来记录自己的学习感悟。新手入坑，好多东西不明所以，搞了好久才把博客搭了起来，特此记录方法。顺便熟悉一下markdown语法。

特别说明

本人用的系统是windows7旗舰版，踩了一个大雷，曾经一度想放弃！

正文开始啦

一、准备阶段

  1. 申请github账号

    首先要有一个github账号吧，申请账号的过程就不多赘述了，过程比较简单。唯一需要说明的是，github网站打开的速度略微有点玄学，我的搜狗浏览器根本打不开，不过用google浏览器就很正常了。

  2. 新建仓库

    申请完github账号后就要new一个repository了，但是每个github账号只能有一个个人页面吧，所以存放个人博客的仓库名字是有要求的（应该是这样吧，我是这样理解的），仓库的名字必须是github账号名.github.io，比如我的github名字是1zhanghan1，所以我的仓库名就要是1zhanghan1.github.io。这个时候，github pages是默认开启的，就不用再做什么操作了。（这里又有一个大雷，后面细说）

  3. 安装git

    然后要下载git了。

    下载地址

    选择适合自己系统的版本就好。下载完成后，点开安装程序一路next就可以了。

    安装完成后，打开cmd命令行输入

    git --version

    出现版本号就是安装成功了。我的是这样的

  4. 安装node.js

    由于我要用hexo搭建我的博客，所以还需要安装node.js，就这，我踩了个大雷。我找到的所有的教程都是直接去官网上下载最新版的，然后我就照着做了，结果。。。安装程序一打开是这样的

    百度一查，版本太高。

    无奈之下，只能自己去找以前的版本。后面找了个12.16.x版本的，结果在后面安装hexo时又版本太低。。。终于，在CSDN找到一个老哥，也是遇到这个问题，他用的是13.14.0版本的。原文链接:

    https://blog.csdn.net/zhangxiaodui/article/details/115209448

    node.js历史版本链接:

    https://nodejs.org/en/download/releases/

    好吧，这就是官网的历史版本下载链接，我竟愚蠢到没有找到！以后可要好好学英语，对英语的敏感度太低了。

    同样，用cmd命令行输入node -v和npm -v查看版本确认一下，出现版本号就是安装成功了。我的是这样。

二、开始搭建

  1. 安装hexo

    从这里开始就是搭建博客啦。我用的是hexo框架，所以要先安装hexo。

    首先在电脑上一个特定的地方（不会忘记的地方）新建一个文件夹，名字任意，例如我的文件夹名是Hexo，鼠标右键选择Git Bash here，会出来一个窗口。输入

    npm install hexo-cli -g

    等待安装完成，然后输入

    hexo -v

    检查版本，显示版本号就ok了。我是这样的。

  2. hexo配置

    在git bash窗口，输入

    hexo init

    等待初始化完成，完成后Hexo文件夹下就会出现很多文件。然后输入

    hexo g 或者 hexo generate生成静态文件

    hexo文件夹下又多了一个public文件夹，这个文件夹就是博客的文件夹

    再输入

    hexo s 或者 hexo server 启动本地服务器

    之后，就可以在默认的地址 http://localhost:4000查看自己的博客了

    按Ctrl+C可以关闭本地预览功能，就可以继续输入代码了

  3. 配置github的SSH

    完成上述步骤后只能在本地看到自己的博客，这显然不是我们想要的。我们需要把自己的博客上传到github仓库才行，这就需要配置SSH KEY。

    然后会有几个问题，第一个是存储的位置，直接回车就是选择默认位置，后两个是设置ssh密码，回车就是不设置密码。所以只要三个回车就OK了。

    成功了的话就可以在默认的位置找到一个.ssh的文件夹，这是个隐藏的文件夹，需要设置显示隐藏文件夹才能看到。点开这个文件夹，会有一个名字是id_rsa.pub的文件，.pub是文件的扩展名，选择用记事本打开，复制里面的所有东西。然后找到github的Settings

    选择SSH and GPG keys，点击New SSH key,在Key一栏粘贴刚刚复制的所有东西，Title任意。

    再点击Add SSH key就OK了，这时候邮箱应该就会收到一个绑定SSH的提醒邮件了。

    repo和branch的冒号后必须有两个空格！type的冒号后好像可以只有一个空格，但是我写成两个空格也没事。

    隐藏雷！！！

    现在github的仓库的默认分支都是main，我找到的教程在branch后写的都是master，因为之前github仓库的默认分支是master，所以这里写main也可以。用master会自动创建一个分支，然后你的博客文件都会上传到这个分支上，导致显示空白。这里也困扰我好久。。。还好之前看了莫烦python的git教程，对分支有了一些了解。这里用master也可以，不过后面要进行一下小设置。

    成功的话，最后会显示

    如果上面_config.yml文件选择的branch是master的话，还需要设置一下github pages 的source。

  5. hexo主题

    刚搭建好的博客用的是hexo的默认主题landscape，看上去丑丑的，那不得换一个好点的吗？

    直接上网址：

    https://hexo.io/themes/

    里面有许多大佬的开源主题，点开可以预览，而且每个主题都注有下载和配置方法。我用的是Chic，比较喜欢它的暗色。

三、开始写博客

还是在原来的git bash窗口

hexo new "文章名"

就可以新建一个文章了。存储地址在Hexo/source/_posts。

文章都是md文件，所以就需要一个markdown编辑器–Typora。直接百度下载安装，没什么好说的。
